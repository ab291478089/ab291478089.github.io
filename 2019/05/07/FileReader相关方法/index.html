FileReader
发布于 2019-05-07   更新于 2019-05-07

今天学习了H5新增属性读取文件的操作，就总结了一下。
FileReader读取文件内容提供的四种方法：

  1. readAsText():读取文本文件（可以使用Txt打开的文件），返回文本字符串，默认编码是UTF-8。
  2. readerAsBinaryString()：读取任意类型的文件，返回二进制字符串。这个方法不是用来给用户看的，而是用于存储文件。例如：读取到的文件内容，获取二进制的数据，传递给后台，后台接受了数据以后，再将数据存在。
  3. readAsDataURL():读取文件以data开头的字符串，实质上就是一段DataURL.
    这种文件是将文件（图像或者能够嵌入到文档的文件格式）嵌入到文档的方案，并将其转换成base64编码的字符串形式，直接存储到url中，因此不需要像服务器发起请求获取数据了。
    》》优化网站的加载速度和效率
  4. abort():中断文件读取。

FileReader还提供了一套事件处理函数

  1. onabort:读取文件中断时触发;
  2. onerror:读取文件失败时触发;
  3. onload:读取文件成功时触发;
  4. onloadend:读取完成时触发,无论成功还是失败;
  5. onloadstart:开始读取时触发;
  6. onprogress:读取文件过程中触发。
